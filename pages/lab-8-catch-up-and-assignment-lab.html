Lab 8: Catch-up and assignment lab

Using the Batch Queue on Raijin

There is no formal lab this week. This is an opportunity for you to go back and look at the previous 3 labs and also to work on your assignment.

So far we have only been running our code on one of the Raijin nodes. In total the Raijin has over 3000 nodes (and 80,000 cores). Six of these are reserved for interactive logins; the remaining nodes are only available via a batch queuing system. (Which of the six interactive nodes are you logged on to? Run the command hostname if unsure.) Go back to the userguide and read the section entitled Job Submission and Scheduling including subsections of Queue Structure, PBSPro Scheduling and PBSPro Basics.

Download the file lab8.tar.gz - this contains an openmp version of the code to compute Pi by numerical integration. We will just use this to demonstrate how the queuing system works.

To submit a job to the queuing system we have to write batch script. An example of this is given in file batch_job. Take a look at this. Lines starting with PBS are commands to the queuing system, informing it of how much resources you require and how your job should be executed. We use one of these lines to set the number of processors you want to use. Very important is the line to limit the walltime:

#PBS -l walltime=00:00:10 Please ensure you limit walltime similarly for any batch job that you use. After all this setup information you run the job (ompbatch N_threads, N_points).

Note the #PBS -lncpus=4 will allocate 4 cores to the job. Thus if you run your job with more threads than cores you are asking the system to oversubscribe threads to cores.

To submit your job to the queuing system, run qsub batch_job.

It will respond with something like

$ qsub batch_job
9485588.r-man2

where 9485588.r-man2 is the id of the job in the queuing system. To see what is happening on the batch queue, run qstat:

aaa444@raijin:~/prac1> qstat

Job id            Name             User              Time Use S Queue
----------------  ---------------- ----------------  -------- - -----
9266041.r-man2    arg.62A          ahf564            285:28:5 R normal-node

                    ---lots of jobs---

9485672.r-man2    batch_job        aaa444                   0 Q express-node

This gives a long list of jobs. In the above the top job is running as indicated by the R in the S column, while my job is queued as indicated by the Q.

When the job has finished you should find a file batch_job.oXXXXX and batch_job.eXXXXX in your directory containing the standard output and error from computation.
